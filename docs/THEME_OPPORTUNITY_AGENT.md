# 테마 기회 발굴 에이전트 설계

기준일: 2026-07-01

## 목적

이 에이전트는 쇼핑몰 경험이 적은 사용자가 다음 과정을 한 번에 수행하도록 돕는다.

```text
테마 선택
→ 허용된 데이터원 수집
→ 공급사 상품 후보 표준화
→ 시장성·수익성·공급·운영 점수화
→ 규정 차단조건 확인
→ 추천 이유·위험·다음 작업 생성
→ 사람의 최종 승인
```

에이전트는 쿠팡이나 다른 판매채널에 상품을 등록하지 않는다.

## 지원 테마

### AGRI_KR

- 농산물
- 지역특산품
- 저장성 농산물
- 건조 농산물
- 신선과일

핵심 게이트:

- 원산지 증빙
- 생산자·품목·중량 표시
- 파손·신선도·배송 정책
- 이미지 권리

### GENERAL_HEALTH_FOOD_KR

- 무가당·저당 일반식품
- 고단백 일반식품
- 식물성 식품
- 통곡물·견과 간식

핵심 게이트:

- 일반식품과 건강기능식품 구분
- 영양성분 표시 근거
- 알레르기·소비기한·보관방법
- 질병 예방·치료 표현 금지

### HEALTH_SUPPLEMENT_KR

핵심 게이트:

- 건강기능식품 일반판매업 신고
- 품목제조신고·제품 식별
- 기능성 원료·기능성 내용
- 섭취 시 주의사항
- 허용 광고문구

판매업 신고 확인 전에는 점수가 높아도 `BLOCKED`다.

## 점수 구조

```text
시장성       35%
수익성       25%
공급안정성   20%
운영난이도   20%
```

규정은 점수에 섞지 않는다.

- `PASS`: 규정상 즉시 차단 사유 없음
- `WARNING`: 사람의 자료 확인 필요
- `BLOCKED`: 승인 불가

## 실행 상태

- `COMPLETED`: 요청한 데이터원이 정상 처리됨
- `PARTIAL`: 일부 데이터원이 실패하거나 설정되지 않았으나 후보 분석은 완료됨
- `FAILED`: 분석할 후보가 없음

## 승인 상태

- `AWAITING_APPROVAL`
- `APPROVED`
- `REJECTED`

승인은 내부 검토 상태일 뿐 외부 판매 등록을 의미하지 않는다.

## 데이터원

### FIXTURE

- 기능 체험용
- 실제 최신 유행이나 판매량을 의미하지 않음
- API 키 없이 사용 가능

### SUPPLIER_CSV

- 사용자가 보유한 공급사 상품파일
- 상품명, SKU, 원가, 배송비, 재고를 후보로 변환

### KAMIS

- `dailySalesList` 최근 도·소매가격
- 공식 인증키와 요청자 ID 필요
- 가격신호이며 검색수요나 실제 온라인 판매량이 아님

### NAVER_TREND

- 검색 관심도 신호
- 판매량으로 표현하지 않음
- 2026년 API HUB 이관에 대응해 endpoint와 인증 헤더를 환경변수화

### FOOD_SAFETY_KOREA

- 제품·신고·영양·인허가 데이터
- 서비스 ID와 검색 필드는 사용 목적에 맞게 설정
- 검색 결과가 있어도 제품 식별자를 사람이 대조해야 함

## 네트워크 경계

- `packages/agent`는 순수 계산·계획·승인 규칙만 포함
- 네트워크 호출은 `apps/local-web/src/agent-sources.ts`에만 존재
- `scripts/check-boundaries.ts`가 core와 agent 패키지의 `fetch`, HTTP, DB import를 차단

## 안전 원칙

1. 로그인·CAPTCHA·접근제어 우회 금지
2. 공식 API 우선
3. API 키 로그 출력 금지
4. API 키 저장소 커밋 금지
5. 데이터원 실패가 전체 실행을 중단시키지 않음
6. 근거·수집시각·fixture 여부 기록
7. AI가 브랜드·원산지·인증·식별자를 추측하지 않음
8. 외부 판매 쓰기 작업 없음
9. `BLOCKED` 후보 승인 금지
10. 모든 승인 후보는 상품 등록 전 진단기로 재검증

## 후속 구현

- 공급사별 컬럼 매핑 프로파일
- KAMIS 품목코드 정밀 매핑
- NAVER API HUB 계약 fixture
- 식품안전나라 서비스별 전용 응답 스키마
- 정책 영향 시뮬레이션
- 승인 후보를 표준 상품 초안으로 변환
- Windows 설치형 앱
