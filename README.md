# 쇼핑플로우 AI

상품 후보를 찾고, 판매 가능성을 진단하고, 등록자료와 쇼핑쇼츠를 만들고, 게시 후 성과를 다시 상품·콘텐츠 개선에 반영하는 **로컬 우선 사람 승인형 AI 커머스 운영 시스템**입니다.

```text
상품 기회 발굴
→ 수익성·규정·등록 준비도 진단
→ 쇼핑쇼츠 캠페인 3종 생성
→ 상품 이미지 기반 9:16 영상 출력
→ 조회·클릭·주문·매출 기록
→ 다음 개선 작업 제안
```

실제 쿠팡·스마트스토어 상품등록, 가격·재고 변경, SNS 자동 게시는 아직 수행하지 않습니다. 모든 외부 쓰기는 후속 단계에서 사람 승인·dry-run·감사 로그를 거쳐 제한적으로 추가합니다.

## 현재 구현된 핵심 기능

### 1. 상품 기회 발굴 에이전트

- 농산물·지역특산품
- 일반 건강식품·헬시플레저 식품
- 건강기능식품
- 데모 데이터와 공급사 CSV 연결
- 시장성·수익성·공급안정성·운영난이도 점수
- 규정 적합성을 `PASS/WARNING/BLOCKED` 게이트로 분리
- 후보별 추천 이유·위험·다음 작업 생성
- 사람이 선택한 후보만 내부 검토 승인
- SQLite 실행·승인 이력

### 2. 상품 등록 전 진단 워크벤치

- CSV 업로드, UTF-8·CP949 처리
- 행 단위 오류 격리와 `PARTIAL` 상태
- Zod 기반 Canonical Product 단일 스키마
- SHA-256 입력 무결성과 결정적 상품 코드
- basis point 정수 가격 계산
- 브랜드·식별자·카테고리·옵션·고시정보 진단
- 쿠팡 비실행 payload preview
- 4축 등록 준비도와 PASS/WARNING/BLOCKED
- 플랫폼 수수료·광고·반품·결제·목표마진 정책
- 반올림 방식·최저 판매가 시뮬레이션
- 화면 인라인 수정 후 즉시 재진단
- JSON·HTML·Issues CSV·Payload JSON 결과

### 3. 하이브리드 쇼핑쇼츠 스튜디오

진단 결과 중 차단되지 않은 상품을 골라 캠페인을 만듭니다.

- `Product → Offer → Campaign → Creative → Performance` 데이터 구조
- 문제 해결형·정보 비교형·구매 전환형 쇼츠 3종
- 15초·30초·45초 장면별 스토리보드
- 첫 2초 후킹 문구
- 장면별 자막·내레이션 초안
- 게시 제목·설명·해시태그·고정댓글
- 광고·협찬·제휴·직접판매 관계 고지
- SRT 자막 다운로드
- 게시문구 TXT 다운로드
- 캠페인 JSON 다운로드
- 상품 이미지 1~10장으로 720×1280 세로 영상 생성
- 브라우저 지원 시 MP4, 미지원 시 WebM 출력
- 이미지 사용권·원산지 증빙 미확인 시 캠페인 승인 차단
- 가격·구성·제조사 미확인 시 경고
- 캠페인과 성과 SQLite 저장

현재 영상 렌더러는 **상품 이미지+움직임+자막 중심의 무음 템플릿형 영상**입니다. TTS·배경음악·서버 FFmpeg 렌더링은 후속 단계입니다.

### 4. 폐쇄형 성과 피드백

캠페인별로 다음 수치를 기록합니다.

- 채널
- 쇼츠 버전
- 조회수
- 클릭수
- 주문수
- 매출
- 광고비

시스템은 클릭률과 주문전환율을 계산해 다음과 같이 제안합니다.

- 조회는 있지만 클릭이 낮음 → 첫 2초 후킹과 구매 이유 개선
- 클릭은 있지만 주문이 낮음 → 가격·배송비·옵션·상세페이지 점검
- 클릭과 주문이 함께 발생 → 성과가 좋은 각도를 유지하고 후킹만 추가 실험

## 요구사항

- Node.js 22.13 이상
- npm
- 영상 렌더링: 최신 Chrome 또는 Edge 권장

## Windows 빠른 실행

GitHub Desktop에서 `main` 브랜치를 Pull한 뒤 저장소 루트의 다음 파일을 더블클릭합니다.

```text
WINDOWS_START.cmd
```

전체 검사:

```text
WINDOWS_CHECK.cmd
```

직접 실행:

```bash
npm ci
npm run typecheck
npm test
npm run check
npm start
```

브라우저:

```text
http://127.0.0.1:3000
```

## 처음 사용하는 순서

1. `상품 기회 찾기`에서 테마와 후보를 확인합니다.
2. `상품 진단`에서 공급사 CSV와 가격정책을 적용합니다.
3. `BLOCKED` 상품은 화면에서 수정하고 다시 진단합니다.
4. `쇼핑쇼츠 만들기`에서 진단 이력과 상품을 선택합니다.
5. 목표 고객·영상 길이·채널·광고 관계를 설정합니다.
6. 원산지 증빙·이미지 사용권·사실정보 확인 여부를 체크합니다.
7. 캠페인을 생성하고 쇼츠 3종을 비교합니다.
8. 상품 이미지를 선택해 세로 영상을 생성합니다.
9. 게시 후 조회·클릭·주문·매출을 입력해 개선 제안을 확인합니다.

## 공식 데이터원

- KAMIS 최근 도·소매가격 Open API
- NAVER Search Trend / NAVER API HUB 교체형 어댑터
- 식품안전나라 Open API
- 공급사 CSV
- 안전한 테마별 fixture

외부 연구는 기본적으로 꺼져 있습니다. `.env.example`을 `.env`로 복사해 필요한 공식 API 키만 설정합니다.

```env
ALLOW_EXTERNAL_RESEARCH=true
KAMIS_CERT_KEY=
KAMIS_CERT_ID=
NAVER_TREND_ENDPOINT=
NAVER_TREND_CLIENT_ID=
NAVER_TREND_CLIENT_SECRET=
FOOD_SAFETY_KEY=
FOOD_SAFETY_SERVICE_ID=
FOOD_SAFETY_QUERY_FIELD=
```

## 데이터와 안전 원칙

- 공식 Open API와 사용자가 제공한 공급사 파일을 우선 사용
- CAPTCHA·로그인·접근제어 우회 금지
- 차단 회피용 프록시 순환 금지
- 검색 관심도를 실제 판매량으로 표현하지 않음
- 브랜드·원산지·인증·효능·식별번호를 추측하지 않음
- 일반식품을 건강기능식품처럼 광고하지 않음
- 이미지 사용권과 원산지 증빙이 없으면 쇼츠 캠페인 승인 차단
- AI 결과는 사실이 아니라 초안이며 게시 전 사람이 검토
- 실제 판매채널·SNS 외부 쓰기 0건

## 저장소 구조

```text
packages/core
  표준상품·가격·규정·진단 순수 로직

packages/agent
  상품 기회 발굴과 외부 데이터 제안

packages/campaign
  캠페인·쇼츠·안전검사·성과 피드백 순수 로직

apps/local-web
  로컬 API, SQLite 저장, 진단·쇼츠 UI, 브라우저 영상 렌더러
```

기본 데이터 파일:

```text
./data/runs.sqlite
```
